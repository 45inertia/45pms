# Project Workflow Research

## Software Development Life Cycle (SDLC)

This is a structured process used to plan, design, develop, test, deploy and maintain software. It
keeps development aligned with business goals and user requirements.

- Provides a clear and organized framework for managing development phases.
- Helps in early detection of defects, reducing overall cost and time.
- Ensures high quality software delivery taht meets user expectations.

### Stages of the Software Development Life Cycle

**Stage 1 - Planning & Feasability Analysis**

This determines whether the project is technically, financialy and operationally fesible.

Activities:
- Feasability analysis
- Cost estimation
- Scheduling
- Resource Planning

Output:
- Project Plan
- Feasibility Report

**Stage 2 - Requirement Specification (SRS)**

The functional and non-functional requirements are documented

Activities:
- Reqauirement Gathering
- Validation
- Documentation

Output:
- Software Requirement Specification (SRS)

**Stage 3 - System Design**

The approved requirements are transformed into a technical blueprint for implementation.

Activities:
- High-Level Design (HLD): Defines system architecture, technology stack, database design, and major
  modules.
- Low-Level Design (LLD): Specifies component logic, APIs, data structures and workflows.

Output:
- Design Document Specification (DDS)

**Stage 4 - Development**

Building the software based off the approved design.

Activities:
- Programming
- Code Reviews
- Unit Testing
- Version Control Management

Output:
- Source Code
- Executable Application

**Stage 5 - Testing**

Testing ensures the software meets requirements and is free from defects before release.

Activities:
- Unit Testing: Verifying Individual Components
- Integration Testing: Ensuring Modules work together
- System Testing: Validating the complete system
- User Acceptance Testing: Confirms requrements are met

Output:
- Test Cases
- Defect Reports
- Quality Metrics

**Stage 6 - Deployment**

Tested software is released.

Activities:
- Production Setup
- Deployment
- Smoke Testing
- CICD: Modern Approach

Output:
- Live Application

**Stage 7 - Maintenance**

Post deployment support to ensure long-term usability

Activities:
- Bug fixes
- Performance Tuning
- Updates
- Feature Enhancements 

Output:
- Patches
- Updates
- New Versions

### Software Development Life Cycle Models
Software Development Models are structured frameworks that guide the planning, execution and
delivery of software projects. They define the sequence of development stages such as requrements,
design, coding, testing, and deployment

**Common Models**

- Waterfall
- Agile
- V Model
- Spiral
- Incremental
- RAD

### Common SDLC Mistakes

- Treating Documentation as the Goal
- Late or inadequate testing
- Ignoring non-functional requrements
- Skipping feedback loops
- Overengineering early
